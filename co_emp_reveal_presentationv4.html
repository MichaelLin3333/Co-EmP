Project Idea

Project idea: affective language understanding

Dominant direction in LLM research

Researchers and developers often emphasize reasoning ability because it directly improves automation of tedious human tasks: coding, planning, summarization, tool use, and problem solving.

Overlooked direction

Natural language also requires detecting, recognizing, and emulating emotion. Affective understanding is part of comprehending language rather than a decorative add-on.

Commercial-scale models such as ChatGPT, DeepSeek, Gemini, and Claude can often display complex emotions because they are trained on enormous corpora. Smaller local models have fewer stored patterns and therefore benefit from explicit affective structure.
04
Goal
Project goal

Investigate the feasibility of modeling emotion dynamics and personality for dialogue-based generation in low-resource scenarios.

Low-resource here means models that can run locally or with modest hardware, rather than relying on the emotional priors of very large commercial systems.

Positioning

What is still missing?

Prior work studies emotion theories, personality theories, affective computing datasets, NLP implementations, and emotion-aware generation for LLMs.

However, these pieces are rarely organized into one lightweight chatbot pipeline where generated dialogue remains consistent with personality and emotional dynamics.

Co-EmP’s contribution is architectural integration: context → affect state estimation → recurrent emotion update → personality-conditioned generation.

Why generate a dialogue-based VAD dataset?

Existing datasets are useful, but they do not directly provide the exact training signal needed for the GRU: turn-level emotional transitions with speaker personality.

Therefore, Co-EmP generates EmoDynamic with scenario, personas, dialogue turns, and VAD labels.

Distillation logic

A strong teacher model generates structured examples. Smaller components learn the lower-dimensional behavior: affect estimation and emotion-state transition.

Dataset Weight

Training weight: not every label source is equal

def source_weight(example):
    """
    - EmoBank gold VAD: high weight
    - custom synthetic VAD: medium/high, depending on quality
    - categorical converted labels: lower weight
    """

    label_source = str(example.get("label_source", "")).lower()
    source = str(example.get("source", "")).lower()

    if "gold_vad" in label_source or "emobank" in source:
        return 1.0

    if "synthetic" in label_source or "custom" in source:
        return 0.4

    if "categorical_to_vad" in label_source:
        return 0.15

Gated Recurrent Unit: tracking emotion over time

We need to keep track of context and mood up to the current utterance. That means recursively updating an emotional state that summarizes previous emotional evidence.

This is analogous to the GRU hidden state ht: information is selectively retained, updated, or reset over time.

LLM generator: prompt as control interface

Few-shot affect guide

Current target affect:
{format_vad(target_vad)}

Interpretation:
- Valence: unpleasant → warm
- Arousal: restrained → urgent
- Dominance: hesitant → firm

Few-shot style guide:
VAD≈(.15,.20,.20):
"I... I don't know what you want me to say."

VAD≈(.15,.85,.80):
"No. Don't twist this around on me. You knew exactly what would happen."

VAD≈(.80,.75,.70):
"Good, then let's fix it now. We still have time."

Generation rules

You are {speaker_name} in a two-person dialogue simulation.
Write only {speaker_name}'s next spoken reply.

Rules:
- Output only the spoken reply.
- No analysis, labels, JSON, or stage directions.
- Do not write the other speaker's line.
- Do not say "valence", "arousal", "dominance", "VAD", or numeric affect values.
- Stay consistent with personality, private context, and shared scenario.
- Avoid long speeches; emulate realistic turn length.

Conclusion

Conclusion

1. Detect

Use a VAD regressor to ground text in continuous affective space.

2. Track

Use GRU dynamics to update emotional state across dialogue turns.

3. Generate

Condition a small LLM with personality and target affect.

Core claim: small LLMs can become more emotionally consistent when affect is modeled explicitly rather than left entirely implicit.
26
Limitations

Current limitations and next steps

1. Semantic content is still weakly captured

The current GRU encoder mainly sees VAD, VAD difference, and personality. It can miss semantic events that should cause emotional change, such as betrayal, confession, loss, or insult delivered in a neutral tone.

Next step: add an appraisal vector so content-level meaning can influence emotional transition.

2. More calibration and alignment are needed

The VAD regressor and GRU both predict VAD, but they do so from different evidence and may not share the same scale. The LLM also needs to interpret GRU target VAD consistently.

Next step: align regressor ↔ GRU and GRU ↔ LLM, potentially with RLHF or DPO.
