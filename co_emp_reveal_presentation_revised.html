Project Idea

Reasoning is not the whole story

Modern model development often prioritizes reasoning, tool use, and task automation.

But language also carries affect: tone, interpersonal stance, confidence, tension, trust, and emotional trajectory.

Large commercial models often learn these patterns implicitly from massive corpora. Smaller local models have less capacity and need more explicit guidance.

Affect
Personality

Co-EmP adds an explicit affective control pathway for low-resource dialogue generation.

Project Goal
Research question
Can we model emotion dynamics and personality to improve dialogue generation in low-resource local LLM settings?

Adding EmoDynamic through synthetic labeling

Why add synthetic data?

  1. Dialogue datasets rarely provide continuous VAD labels.
  2. Generated dialogues can include persona, VAD, and interaction history.
  3. The logic is similar to distillation: a stronger teacher model provides structured supervision to a smaller downstream model.

VAD Training

Dataset weighting

Reasoning

Different sources have different reliability for VAD supervision.

  • Gold VAD: direct supervision
  • Synthetic VAD: useful but teacher-generated
  • Categorical-to-VAD: approximate mapping
def source_weight(example):
    label_source = str(example.get("label_source", "")).lower()
    source = str(example.get("source", "")).lower()

    if "gold_vad" in label_source or "emobank" in source:
        return 1.0

    if "synthetic" in label_source or "custom" in source:
        return 0.4

    if "categorical_to_vad" in label_source:
        return 0.15

    return 0.2

GRU Encoder

Section 2 — why a recurrent model?

A dialogue’s emotional state is not reset every turn.

The next emotional state depends on the previous state, the other speaker’s current utterance, and the responding speaker’s personality.

A GRU is useful because it maintains a hidden state that is updated step-by-step across a sequence.

LLM Generator

Section 3 — guiding a local LLM

Important prompt controls

  • Expose target affect as an interpretation guide, not as text to repeat.
  • Use few-shot examples to map VAD regions into natural speaking styles.
  • Keep strict output rules: one reply, no labels, no analysis, no VAD terms.
  • Condition on public scenario, speaker personality, and private speaker context.
Affect guidance:
- Low valence: hurt, angry, bitter, afraid
- High arousal: tense, urgent, defensive
- Low dominance: hesitant, apologetic, uncertain
- High dominance: firm, decisive, confrontational

Few-shot anchors:
(0.15, 0.20, 0.20)
"I... I don’t know what you want me to say."

(0.15, 0.85, 0.80)
"No. Don’t twist this around on me."

Generation rule:
Write only the next spoken reply.
LLM Generator

Prompt rules: compact form

System role

You are speaker_name in a two-person dialogue simulation.

Stay consistent with personality, private context, and shared scenario.

Use affect guidance internally, but do not mention VAD or numeric values.

Conclusion

Conclusion

What was built

  • VAD regressor for affect state estimation
  • GRU encoder for emotion dynamics
  • LLM prompting layer for target affect control
  • Gradio demo with comparison modes

Main claim

Small local LLMs can be made more emotionally consistent when affect and personality are represented explicitly rather than left entirely implicit inside the model.

Limitations

Current limitations and next steps

1

Semantic appraisal is still weak

At present, the GRU encoder mainly processes VAD history and personality. It may fail to capture semantic events that strongly change emotion, such as betrayal, accusation, apology, or sudden loss. This can be improved by adding an appraisal vector for goal relevance, norm violation, agency, controllability, and relationship impact.

2

More calibration is needed

The VAD regressor and GRU encoder both predict VAD, but from different signals and distributions. Their outputs need stronger alignment. A second alignment is also needed between the LLM’s practical interpretation of VAD and the GRU’s target VAD. RLHF or DPO could be used for this calibration.
